## WB Учёт v2.0 — что изменилось

### Архитектура (9 файлов + HTML)

| Файл | Назначение |
|------|-----------|
| `00_Config.gs` | Все константы, WB_API домены, схемы листов с описаниями |
| `01_Utils.gs` | `wbRequest()`, форматирование дат, запись/чтение листов |
| `02_Settings_11_Logs.gs` | Настройки с группировкой, система логов |
| `03_06_...gs` | Артикулы, остатки WB, заказы, продажи |
| `07_08_...gs` | Финансы, баланс продавца, поставки FBW |
| `09_Reports.gs` | ДДР с ДРР%, расходы |
| `12_Menu.gs` | Меню, Sidebar, `loadAll`, белый список функций |
| `Sidebar.html` | UI-панель: 4 вкладки (действия, листы, настройки, справка) |
| `DEV_GUIDE.gs` | Инструкция разработчика прямо в коде |

### Ключевые улучшения

**Даты** — все даты в листах теперь в формате `дд.мм.гггг чч:мм:сс`. В API WB отправляются в ISO через `parseDateToIso()`.

**Новые листы** — добавлены: `ОСТАТКИ_WB`, `БАЛАНС`, `ЛОГИ`.

**Логи** — каждый запуск пишет строку: время начала/конца, длительность в сек, статус OK/ERROR, кол-во строк. Ошибки подсвечиваются красным.

**HTML Sidebar** — 4 вкладки: кнопки запуска всех функций, статус листов со счётчиком строк, форма настроек, справка по всем листам и токенам.

**Адаптация к изменениям API** — все домены в одном месте `WB_API{}`. При смене домена WB меняешь одну строку. Паузы берутся из `rateLimit.sleepMs`.

**Масштабируемость** — добавление нового модуля по инструкции в `DEV_GUIDE.gs`: 8 чётких шагов с примером кода.

### Важно по токенам WB
Финансовый отчёт (`/api/v5/supplier/reportDetailByPeriod`) требует токен категории **Статистика**, а баланс (`/api/v1/account/balance`) — токен категории **Финансы**. Создай отдельные токены в кабинете WB под каждую категорию.
